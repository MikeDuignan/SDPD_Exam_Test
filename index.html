Question 1 (30 marks)

Write a Java program that simulates a Fuel Pump System using a custom exception.

  1. Create a custom exception class named InvalidFuelAmountException that extends the Exception class. It should include a constructor that accepts a custom error message.
  2. In your main program, use a loop to continuously prompt the user to enter the number of litres they wish to purchase (as a double).
  3. If the user enters a value below 1 or above 100, throw the InvalidFuelAmountException with the message "Error: Invalid amount! You must purchase between 1 and 100 litres.".
  4. Use a try-catch block to catch the exception and display the error message. The loop should continue asking for input if this exception occurs.
  5. If a valid amount is entered, calculate the total cost at a rate of €1.85 per litre, display the amount of fuel and the total cost (formatted to 2 decimal places), and terminate the loop.
  6. Also handle InputMismatchException to prevent the program from crashing if the user enters non-numeric text (e.g., letters). Display a generic error message like "Error: Please enter a valid number." and allow the user to try again. (Remember to clear the scanner buffer if this occurs).

Additional Requirements:

  • Use appropriate loops for input and processing.
  • Use meaningful variable names and include comments.

Question 2 (70 marks)

Write a Java program that implements a Vehicle Fleet Management System. This program must demonstrate inheritance, constructors, and static fields/methods.

Class Requirements:

1. Base Class (Vehicle) (25 marks):

  • Create a class called Vehicle with the following protected fields: make (String) and registrationNumber (String).
  • Add a private static field called totalVehiclesRegistered (int) and initialize it to 0. This will track how many vehicles have been created across all subclasses.
  • Create a parameterized constructor that takes make and registrationNumber. Inside this constructor, increment the totalVehiclesRegistered static field.
  • Create a regular method void displayVehicleDetails() that prints the make and registration number to the console.
  • Create a public static method called getTotalVehicles() that returns the current value of the totalVehiclesRegistered field.

2. Subclass 1 (Car) (15 marks):

  • Create a class called Car that inherits from Vehicle.
  • Add private fields for numDoors (int) and fuelType (String).
  • Create a constructor that accepts make, registrationNumber, numDoors, and fuelType. Use the super() keyword to call the base class constructor.
  • Override the displayVehicleDetails() method to print the car's specific details (number of doors, fuel type) in addition to calling the superclass's displayVehicleDetails() method.

3. Subclass 2 (Motorcycle) (15 marks):

  • Create a class called Motorcycle that inherits from Vehicle.
  • Add a private field for engineSize (int), representing the engine capacity in cc.
  • Create a constructor that accepts make, registrationNumber, and engineSize. Use the super() keyword to call the base class constructor.
  • Override the displayVehicleDetails() method to print the motorcycle's specific details (engine size) in addition to calling the superclass's displayVehicleDetails() method.

Program Requirements (Main Method) (15 marks):

  • Call the static method Vehicle.getTotalVehicles() and print the initial number of vehicles (this should be 0).
  • Create one Car object with sample data (e.g., Make: "Toyota", Reg: "191-G-1234", Doors: 4, Fuel: "Petrol").
  • Create one Motorcycle object with sample data (e.g., Make: "Yamaha", Reg: "221-D-5678", Engine: 600).
  • Call the displayVehicleDetails() method on both the Car and the Motorcycle objects to output their information.
  • Call the static method Vehicle.getTotalVehicles() again and print the final total number of vehicles registered (this should be 2).
